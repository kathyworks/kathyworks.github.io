Building a full-stack MERN web app: [Ch. 1] The setup

July 8, 2024
Getting started.

Brief Backstory​

I am beginning the process of turning my blog into a self-hosted web app, so I thought I would take this opportunity to create a step-by-step documentation / tutorial as I go.

The Tech Stack​

The MERN stack is what I'll be using in this project.

  • MongoDB
  • Express
  • React
  • NodeJS

The internet already has abundant resources that explain what a MERN stack is and what it is (or is not) capable of, so I won't go into too much detail here. Quite simply put, both the frontend and backend sides of the code will primarily be written in JavaScript, making this stack highly versatile for small dev teams (or even one-person bands).

My Setup​

The beauty of building a web app is you can easily get started in virtually any environment. But just for the sake of succinctness, I am listing what I currently use in this project for reference:

  • macOS
    (Windows or Linux would work just as well here)
  • Terminal
  • Visual Studio Code
  • [Optional] A clicky mechanical keyboard

Tone​

Whilst this series is essentially just me documenting my own steps, I decided to type everything out as if I am instructing another developer (perhaps you?) on how to do it-just so the chapters will flow more naturally.

Let's dive right in.

Creating a NodeJS Application​

Let's start by building out a backend first. In our case, it will be a NodeJS app.

If you are reading this series, chances are you would already have NodeJS and npm installed. To check the versions you have, type this command in Terminal:

node -v && npm -v

In case NodeJS has not been installed on your local machine yet, it can be downloaded from https://nodejs.org.

When you install NodeJS this way, npm will automatically get installed as well.

Folder Structure​

My preferred top-level project folder structure is as follows, where "Kathy Works" is the name of my blog:

~/kathy_works
├── /backend/
└── /frontend/

Feel free to name your own project anything you want.

In this chapter, I'll only be working on the /backend/ part. We will get to the /frontend/ in future chapters.

To create the desired folders:

mkdir kathy_works
cd kathy_works
mkdir backend
cd backend

package.json​

In modern node development, a package.json file is crucial for installing node packages in the project.

To scaffold the /backend/ with a default package.json, run the following command:

npm init -y

.gitignore​

A .gitignore is highly recommended if you use git for version control. As its name suggests, files and folders listed in the .gitignore file will be ignored by the GitHub repository.

The dot in front of the filename means the file is a hidden one.

Installing Packages​

For this project, we will be installing several necessary packages:

  • Express
    The NodeJS framework we'll use to create API endpoints, which will act as communicative bridges between our frontend and our database.

  • Mongoose
    A package for us to connect to our MongoDB database.

  • Dotnev
    It is commonly used to store project secrets or special configurations that we do not want to go onto a live database.

  • Nodemon
    A great development tool, which is capable of restarting our backend server as soon as we make any changes in the codebase.

To install the first 3 packages, run:

npm i express mongoose dotenv

As for nodemon, since we won't need it in production, we will install it as a dev dependency via the --save-dev flag:

npm i --save-dev nodemon

Now if we open up package.json, we should see that the packages have been installed appropriately in either "dependencies" or "devDependencies".

"dependencies": {
    "dotenv": "^16.4.5",
    "express": "^4.19.2",
    "mongoose": "^8.4.5"
},
"devDependencies": {
    "nodemon": "^3.1.4"
}

Running the Project​

The default main javascript file in package.json is index.js. However, the more common convention in backend node development is to name it server.js, so we'll create it and give it just one line (for now) in Visual Studio Code:

console.log("Hello world! I am your server.");

Next, in package.json, update the "main" and "scripts" sections accordingly to reflect our changes:

"main": "server.js",
"scripts": {
    "start": "node server.js",
    "dev": "nodemon server.js"
},

At this point, our folder structure should look similar to this:

/kathy_works
├── /backend
│   ├── .gitignore
│   ├── /node_modules/
│   ├── package-lock.json
│   └── package.json
└── /frontend/

As we are just getting started, we are not quite ready to deploy our code to production yet.

Let's run in development for the time being. In Terminal:

npm run dev

If everything works properly, you should see:

[nodemon] starting `node server.js`
Hello world! I am your server.

Congratulations!
